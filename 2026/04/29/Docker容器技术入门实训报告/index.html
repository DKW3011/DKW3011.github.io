Docker容器技术入门实训报告

一、实训目的

本次实训围绕 Docker 容器技术展开，核心目标如下：

  1. 掌握华为云环境下 Docker 服务的安装、启动与状态验证方法，理解容器技术的核心优势。

  2. 熟练使用 Docker 命令完成镜像拉取、容器创建、运行、进入与生命周期管理（启动 / 停止 / 删除）。

  3. 掌握 Dockerfile 构建自定义镜像的流程，理解镜像分层构建原理，实现应用的容器化打包。

  4. 掌握容器端口映射、日志查看、数据交互等实用操作，理解容器与宿主机的网络、文件交互逻辑。

  5. 理解容器的资源限制、环境变量配置、数据卷挂载等进阶功能，掌握容器应用的测试与故障排查方法。

二、实训环境

云平台 华为云弹性云服务器 ECS（实验沙箱环境）
操作系统 CentOS
核心软件 Docker CE（社区版）、Linux 命令行工具
操作工具 华为云 CloudShell 远程登录工具、Chrome 浏览器

三、实训内容

本次实训以 Docker 容器的完整使用流程为主线，核心内容分为六大模块：

  1. 实验环境准备：登录华为云实验环境，完成 Docker 服务的安装与基础配置。

  2. Docker 基础操作：镜像拉取、容器创建与运行、容器生命周期管理。

  3. 容器交互与数据管理：进入容器内部操作、安装工具、查看容器日志与资源占用。

  4. 自定义镜像构建：通过 Dockerfile 构建自定义应用镜像，理解构建流程与优化要点。

  5. 容器网络与端口映射：配置容器端口映射，实现外部访问容器内的应用服务。

  6. 容器进阶管理：容器资源限制、数据卷挂载、镜像推送与容器应用的综合测试。

五、实训结论

  1. 实训成果
    本次实训成功完成了 Docker 容器技术的全流程实践：Docker 服务安装配置正常，可拉取、管理公共镜像；能熟练创建、运行、管理容器，实现端口映射与外部访问；通过 Dockerfile 成功构建了自定义应用镜像，容器化应用可正常提供服务；掌握了容器数据管理、资源限制与故障排查方法，所有实训任务均顺利完成。

  2. 知识与技能收获

    • 理解了 Docker 容器技术的核心原理，掌握了镜像、容器、仓库的基础概念与操作命令。

    • 熟练掌握了 Docker 的基础操作：镜像拉取 / 构建、容器创建 / 启停 / 删除、日志查看与资源监控。

    • 掌握了 Dockerfile 的编写规则与自定义镜像构建流程，理解了镜像分层构建的优势。

    • 理解了容器的网络与数据管理机制，掌握了端口映射、数据卷挂载的配置方法。

  3. 问题与解决经验

    • 端口访问失败：容器端口映射配置正确，但外部无法访问，排查发现安全组未放行宿主机端口，配置安全组规则后恢复正常。

    • 自定义镜像构建失败：Dockerfile 语法错误或文件路径配置错误，通过检查构建日志修正语法问题，确认文件路径后构建成功。

    • 容器内命令无法执行：容器缺少相关工具包，通过docker exec进入容器安装依赖工具解决。

  4. 实训总结
    本次实训系统掌握了 Docker 容器技术的使用流程，从基础操作到自定义镜像构建，再到容器应用的部署与管理，深入理解了容器化技术在应用部署中的优势。通过本次实训，为后续微服务架构、DevOps 流程中的容器化部署打下了坚实基础。
