当本地服务器端口成功加载出localhost:4000的预览页面，看着终端里滚动的渲染日志，第一次用代码搭建出属于自己的线上空间，那种成就感比调试通算法作业时更加强烈。这一周从技术选型到成功部署，就像完成了一门超有实践意义的课程设计。

技术选型： 为啥选择Hexo+GitHub ？

作为计算机专业学生，选建站工具自然带着专业视角。对比过 WordPress 的动态架构和 Jekyll 的 Ruby 生态后，Hexo 的技术栈更符合我的学习方向：基于 Node.js 的静态站点生成器，不仅部署轻便，还能顺便巩固 JavaScript 和前端工程化知识。

实际体验后发现它的优势远超预期：Markdown 原生支持完美适配程序员的写作习惯，插件化架构像搭组件库一样扩展功能，Git 集成部署更是我们日常开发的熟面孔。用hexo init命令初始化项目时，看着自动生成的目录结构，仿佛看到了一个小型前端项目的骨架。More info: Generating

我的技术自留地规划

这个网站对我来说是专业成长的 “线上实验室”：

  • 记录专业课的算法笔记与实现代码，用代码块展示解题思路

  • 分享前端开发的学习心得，从 HTML 基础到框架应用逐步深入

  • 存放课程设计的项目文档，包括需求分析和架构设计

  • 整理编程错题集，像调试 bug 一样记录知识盲区

就像给项目写 README 文档，我在网站首页添加了简洁的自我介绍和技术栈展示。作为计算机学生，终于有了一块能自由展示技术成长的空间，这种用专业技能创造价值的感觉，让我对未来的学习充满动力。
